#### Why use package-manager

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


CommonJS 出现，使 node 环境下，JS 代码可以更加细粒度的规划: 类，函数，对象等

为了解决前端开发过程中遇到的常见问题，比如: 加密，提供常见的工具方法，模拟数据等

前端社区涌现了大量的第三方库，这些库均使用 CommonJS 书写而成，易于使用


使用第三方库，过程繁琐

    - 进入官网或 git 主页

    - 找到并下载响应的版本 

    - 拷贝到工程文件目录中

    - 若遇到同名库，需要修改名称


包管理工具所解决的问题

1) 当所下载的库与其他库产生较多链式依赖，手动下载无法便捷的设置文件关系

2) 开发环境中安装的大量的第三方库，如何在生产环境中还原并区分

3) 如何更新库

4) 自己开发的库，如何在下一个开发中使用


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```

#### npm

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


/**
 * node package mamager                     // node 包管理器
 * 
 * 
 * 运行在 node 环境中，使开发者用简单的方式完成包的管理: 安装，更新，卸载，上传等
 */

npm 之所以运行在 node 环境中，而不是浏览器环境

根本原因: 因为浏览器环境无法提供下载，删除，读取本地文件等功能

而 node 属于服务器环境，没有 浏览器的种种限制，理论上 node 可以完全掌控运行本地计算机


//-------------------------------------------------------------------------------------------------------------------//


// npm 由三部分组成

/**

 * 
 * 
 */

根据 npm 的规范，将开发的完整包上传到数据库中( eg: github )，以便于其他开发者使用
```
